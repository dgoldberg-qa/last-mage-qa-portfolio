# Early access mobile game QA testing

## About

This repository contains my QA portfolio project based on testing the early access Android game Last Mage.

The project demonstrates  the complete QA testing process, from planning and test documentation to bug reporting and the final test summary.

## Testing activities

The following QA activities were performed:

- Checklist creation
- Test case design
- Exploratory testing
- Bug reporting
- Improvement suggestions
- Open questions
- Test summary report

## Test environment

- **Game:** Last Mage (early access)
- **Platform:** Android
- **Device:** Samsung Galaxy A16
- **Operating system:** Android 15
-  **Game version:** 0.5.2046

## Repository structure

- **Checklist** - Functional and exploratory testing checklist
- **Test_cases** - Test cases covering core gameplay features
- **Bug_reports** - Reproducible bugs identified during testing
- **Improvements** - Usability and UX improvement suggestions
- **Questions** - Gameplay mechanics requiring clarification from the developer
- **Test_summary** - Final testing summary and conclusions

## Project summary

During testing:

- 109 checklist items were executed
- 12 test cases were created
- 2 reproducible bugs were reported
- 8 improvement suggestions were documented
- 2 open questions were prepared for clarification

The testing focused on core gameplay mechanics, user interface, inventory management, progression system, advertisements, performance and overall user experience.

## Objective

The goal of this project was to practice manual QA testing by documenting the testing process and providing structured feedback for an early access mobile game.
